Overview

shinyTooltips is a lightweight, modern library for adding interactive tooltips to Shiny applications. It provides a simple yet powerful API for creating customizable, accessible tooltips with minimal configuration.

Key Features

  • Multiple Placements: Tooltips can appear above, below, to the left, or to the right of target elements
  • Multiple Triggers: Support for hover, click, and focus triggers
  • Rich Content: Use HTML, text, or complex Shiny tagLists as tooltip content
  • Customizable Styling: Apply custom CSS styles, predefined sizes, or use theme customization
  • Animations: Smooth fade or scale animations
  • Interactive Mode: Create tooltips with inputs and interactive controls
  • Smart Positioning: Automatically adjusts position to avoid viewport overflow
  • No Dependencies: Pure JavaScript, no jQuery required

What Problems Does It Solve?

Shiny tooltips are commonly needed for:

  • Providing contextual help and hints for form inputs
  • Explaining button functionality without cluttering the UI
  • Displaying additional information on hover
  • Creating interactive tooltips with forms or controls
  • Improving user experience with minimal space overhead

Getting Started

Installation

Install directly from GitHub using the devtools package:

devtools::install_github("yourusername/shinyTooltips")

Basic Setup

Follow these steps to get started with shinyTooltips:

  1. Load the library:
    library(shinyTooltips)
  2. Initialize in your UI:

    Add use_shiny_tooltips() to your Shiny UI to load the required JavaScript and CSS files.

    ui <- fluidPage( use_shiny_tooltips(), # ... rest of your UI )
  3. Add tooltips to elements:

    Use the tooltip() function to attach tooltips to elements.

    tooltip("myButton", "Click to submit") actionButton("myButton", "Submit")

First Complete Example

Here's a complete Shiny app with tooltips:

library(shiny) library(shinyTooltips) ui <- fluidPage( use_shiny_tooltips(), h1("Welcome!"), # Add tooltip to a button tooltip("btn", "Click me!"), actionButton("btn", "Click Me"), br(), br(), # Add tooltip to text input tooltip("email", "Enter your email address", placement = "right", trigger = "focus"), textInput("email", "Email:", ""), # Display output textOutput("result") ) server <- function(input, output, session) { output$result <- renderText({ if (input$btn > 0) { return(paste("Button clicked:", input$btn, "times")) } "Button not clicked yet" }) } shinyApp(ui, server)

API Reference

use_shiny_tooltips()

Initialize the tooltip system in your Shiny UI.

use_shiny_tooltips(theme = NULL)

Parameters

Parameter Type Default Description
theme list NULL Named list of CSS properties for custom theme (e.g., list(background = "#333", color = "#fff"))

Usage

# Basic initialization ui <- fluidPage( use_shiny_tooltips(), # ... rest of UI ) # With custom theme ui <- fluidPage( use_shiny_tooltips(theme = list( background = "#2c3e50", color = "#fff", borderRadius = "8px" )), # ... rest of UI )

tooltip()

Attach a tooltip to a target element. Call this before the target element in your UI.

tooltip( target, content, placement = "top", trigger = "hover", animation = "fade", delay = 0, offset = 10, size = "medium", width = NULL, maxWidth = NULL, style = NULL, interactive = FALSE )

Parameters

Parameter Type Default Description
target character ‚Äî ID of the target element (required)
content character/tagList ‚Äî Tooltip content (text, HTML, or tagList)
placement character "top" Tooltip position: "top", "bottom", "left", "right"
trigger character "hover" Trigger type: "hover", "click", "focus"
animation character "fade" Animation style: "fade", "scale"
delay numeric 0 Delay in milliseconds before showing tooltip
offset numeric 10 Distance in pixels between tooltip and target
size character "medium" Predefined size: "small", "medium", "large", "auto"
width character NULL Fixed width (e.g., "300px", "50%")
maxWidth character NULL Maximum width (e.g., "500px")
style character NULL Custom CSS styles (e.g., "background: #333; color: #fff;")
interactive logical FALSE If TRUE, tooltip stays visible when hovering over it

Return Value

Returns HTML code that should be placed in your UI. Call this before the target element.

Examples

1. Basic Button Tooltip

The simplest example - a tooltip on a button:

library(shiny) library(shinyTooltips) ui <- fluidPage( use_shiny_tooltips(), tooltip("btn", "This is a helpful message"), actionButton("btn", "Hover over me") ) server <- function(input, output, session) {} shinyApp(ui, server)

2. Form with Contextual Help

Add tooltips to form inputs for guidance:

library(shiny) library(shinyTooltips) ui <- fluidPage( use_shiny_tooltips(), h2("Sign Up Form"), tooltip("email", "Enter a valid email address", placement = "right", trigger = "focus"), textInput("email", "Email:"), br(), tooltip("password", "Minimum 8 characters", placement = "right", trigger = "focus"), passwordInput("password", "Password:"), br(), actionButton("submit", "Sign Up") ) server <- function(input, output, session) {} shinyApp(ui, server)

3. Rich Content with Tags

Create complex, formatted tooltip content:

library(shiny) library(shinyTooltips) ui <- fluidPage( use_shiny_tooltips(), tooltip( "info", tagList( tags$h4("What is this?"), tags$p("This is a tooltip with:"), tags$ul( tags$li("Rich formatting"), tags$li("HTML elements"), tags$li("Custom styling") ) ), width = "250px" ), actionButton("info", "Info") ) server <- function(input, output, session) {} shinyApp(ui, server)

4. Interactive Tooltip with Input

Tooltip with interactive form controls:

library(shiny) library(shinyTooltips) ui <- fluidPage( use_shiny_tooltips(), tooltip( "settings", tagList( h4("Quick Settings"), textInput("setting_name", "Name:", width = "100%"), selectInput("setting_theme", "Theme:", choices = c("Light", "Dark"), width = "100%"), actionButton("save", "Save", style = "width: 100%;") ), width = "280px", interactive = TRUE, trigger = "click" ), actionButton("settings", "Open Settings"), br(), br(), textOutput("settings_result") ) server <- function(input, output, session) { output$settings_result <- renderText({ if (input$save > 0) { paste("Settings saved: Name =", input$setting_name, ", Theme =", input$setting_theme) } else { "No settings saved yet" } }) } shinyApp(ui, server)

5. All Placements Showcase

Demonstrate all four tooltip placements:

library(shiny) library(shinyTooltips) ui <- fluidPage( use_shiny_tooltips(), h2("Tooltip Placements"), div(style = "display: flex; gap: 20px; justify-content: center; margin: 40px 0;", div( tooltip("top", "Top placement", placement = "top"), actionButton("top", "Top") ), div( tooltip("right", "Right placement", placement = "right"), actionButton("right", "Right") ), div( tooltip("bottom", "Bottom placement", placement = "bottom"), actionButton("bottom", "Bottom") ), div( tooltip("left", "Left placement", placement = "left"), actionButton("left", "Left") ) ) ) server <- function(input, output, session) {} shinyApp(ui, server)

6. Styled Tooltips

Apply custom styles to tooltips:

Best Practices

Content Guidelines

Tip: Keep tooltip content concise. A single sentence or short phrase is ideal. If you need more space, use rich content with tagList.
  • Use clear, actionable language
  • Keep text to 1-2 lines when possible
  • Avoid repeating button or label text
  • Don't hide critical information in tooltips only
  • Use simple language, avoid jargon

Placement Guidelines

Note: The library automatically adjusts tooltip position to keep it in the viewport. However, planning placements can improve perceived layout.
  • Use placement = "bottom" for top-of-page buttons
  • Use placement = "right" for left-side inputs
  • Avoid placements that cause tooltips to overlap page edges
  • Be consistent with placement across related elements

Trigger Guidelines

  • Use trigger = "hover" for buttons and link-like elements
  • Use trigger = "focus" for form inputs (text, password, select)
  • Use trigger = "click" for important information users should explicitly access
  • Use trigger = "click" for interactive tooltips with forms

Styling Guidelines

Tip: Use custom styles to match your app's design system. Consistent styling improves usability.
  • Use use_shiny_tooltips(theme = ...) to set a global theme
  • Apply consistent colors across your app's tooltips
  • Ensure sufficient contrast for accessibility (WCAG AA minimum)
  • Use size variants to match your content length
  • Test on different screen sizes and devices

Performance Considerations

  • Lazy load tooltips for many elements (consider pagination)
  • Use simple content for better performance
  • Avoid complex reactive expressions inside tooltips
  • Set appropriate delays to prevent flickering

Accessibility Guidelines

Important: Tooltips should enhance, not replace, accessibility features.
  • Ensure keyboard navigation works (use trigger = "focus")
  • Maintain sufficient color contrast
  • Don't rely solely on tooltips for critical information
  • Test with screen readers
  • Ensure tooltip text is readable at 200% zoom

Mobile Considerations

  • Test tooltip visibility on small screens
  • Consider trigger = "click" instead of "hover" on mobile
  • Ensure tooltips don't block important content
  • Use interactive = TRUE carefully on touch devices

Real-World Example Pattern

Here's a recommended pattern for form tooltips:

# Good pattern for form help tooltips div(class = "form-group", div( tooltip("email", "Enter a valid email (e.g., user@example.com)", placement = "right", trigger = "focus"), textInput("email", label = "Email Address:", width = "100%") ) )

FAQ & Troubleshooting

Frequently Asked Questions

Q: Do I need to call tooltip() before or after the target element?
A: Call tooltip() immediately before the target element in your UI. The order matters!
# Correct tooltip("btn", "Help text") actionButton("btn", "Click me") # Incorrect actionButton("btn", "Click me") tooltip("btn", "Help text")
Q: How do I create an interactive tooltip with forms?
A: Set interactive = TRUE and use tagList for rich content:
tooltip("btn", tagList( textInput("name", "Name:"), actionButton("submit", "Submit") ), interactive = TRUE, trigger = "click" ) actionButton("btn", "Settings")
Q: My tooltip is not appearing. What should I do?
A: Check these common issues:
  1. Did you call use_shiny_tooltips() in your UI?
  2. Is the target element ID correct? (Must match exactly)
  3. Is tooltip() placed before the target element?
  4. Does the target element exist in the DOM?
  5. Check your browser console for JavaScript errors
Q: Can I use shinyTooltips with renderUI or dynamic content?
A: Yes, but add use_shiny_tooltips() only once in the main UI. For dynamically added tooltips, ensure the target elements are rendered and tooltip definitions are loaded correctly.
Q: How do I style tooltips to match my app theme?
A: Use the theme parameter in use_shiny_tooltips():
use_shiny_tooltips(theme = list( background = "#2c3e50", color = "#ffffff", borderRadius = "8px", fontSize = "14px" ))
Q: What's the difference between size and width parameters?
A: Use size for predefined dimensions, width for exact values:
# Predefined sizes tooltip("btn", "Short", size = "small") tooltip("btn", "Medium", size = "medium") tooltip("btn", "Long content here", size = "large") # Custom size tooltip("btn", "Exactly 350px", width = "350px") tooltip("btn", "Max 500px", maxWidth = "500px")

Troubleshooting Common Issues

Issue: Tooltip appears outside the viewport
The library automatically adjusts position. If it still occurs:
  • Try different placement values
  • Reduce offset value
  • Use smaller size or width
  • Increase viewport by using responsive design
Issue: Tooltip content is not rendering HTML
Use tagList from htmltools for rich content:
# This won't work (HTML not rendered) tooltip("btn", "Bold text") # This works (proper HTML) tooltip("btn", tagList( tags$b("Bold text") ))
Issue: Interactive tooltip closes unexpectedly
When using interactive = TRUE:
  • Ensure tooltip stays visible when hovering over it
  • Check that content doesn't disappear on scroll
  • Test with trigger = "click" for better control
  • Set appropriate delay value
Issue: Styling not applying to tooltip
CSS properties must be in JavaScript camelCase format:
# Wrong (hyphens) style = "background-color: #333;" # Correct (camelCase) style = "backgroundColor: #333;"
